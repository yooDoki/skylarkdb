# SkylarkDB 数据导出使用指南

## Redis 数据导出

### 如何使用

1. 连接到 Redis 数据库
2. 在左侧键列表中点击选择一个键
3. 在右侧详情面板中，点击 **JSON** 或 **TXT** 按钮
4. 导出成功后会显示文件保存路径

### 导出位置

文件默认保存到系统的临时目录：
- **macOS/Linux**: `/tmp/`
- **Windows**: `C:\Users\<用户名>\AppData\Local\Temp\`

### 导出格式

#### JSON 格式
```json
{
  "key": "user:123",
  "type": "string",
  "value": "John Doe"
}
```

#### TXT 格式
```
Key: user:123
Type: string
Value:
John Doe
```

### 支持的键类型

- ✅ String（字符串）
- ✅ Hash（哈希）
- ✅ List（列表）
- ✅ Set（集合）
- ✅ Zset（有序集合）

### 注意事项

1. 导出 large 数据时可能需要几秒钟
2. 如果键不存在，会显示错误提示
3. 导出文件名会自动处理特殊字符（`:`、`\`、`/` 会被替换为 `_`）

## MySQL 数据导出

### 如何使用

1. 连接到 MySQL 数据库
2. 选择一个表查看数据
3. 点击工具栏的 **导出** 按钮
4. 选择导出格式和保存位置

### 导出格式

- **JSON**: 结构化数据，易于程序处理
- **CSV**: 表格格式，可用 Excel 打开
- **SQL**: INSERT 语句，可直接导入数据库

---

## 批量导出（Redis）

### 如何使用

1. 点击 **批量** 按钮进入批量模式
2. 勾选要导出的多个键
3. 点击 **导出 JSON** 按钮
4. 所有选中的键会分别导出为单独的文件

### 批量导出位置

所有文件会保存到同一个目录：`/tmp/`

---

## 故障排除

### 导出失败

**问题**: 点击导出按钮后提示失败

**解决方案**:
1. 检查是否已选择键
2. 检查连接是否正常
3. 查看错误提示信息
4. 如果是权限问题，尝试以管理员身份运行

### 找不到导出文件

**问题**: 导出成功但找不到文件

**解决方案**:
1. 查看导出成功提示中的完整路径
2. 在文件管理器中打开该路径
3. macOS 用户可以在 Finder 中使用 `Cmd+Shift+G` 跳转到 `/tmp/`
4. Windows 用户可以在资源管理器地址栏输入 `%TEMP%`

### 导出文件为空

**问题**: 导出成功但文件内容为空

**解决方案**:
1. 检查键是否存在
2. 检查键的类型是否支持导出
3. 重新尝试导出
